# <a name="list-of-prerequisite-checks-for-system-center-configuration-manager"></a>Liste der Voraussetzungsprüfungen für System Center Configuration Manager

*Gilt für: System Center Configuration Manager (Current Branch)*

In den folgenden Abschnitten werden die verfügbaren Voraussetzungsprüfungen ausführlich erläutert.

Informationen zum Verwenden der Voraussetzungsprüfung finden Sie unter [Voraussetzungsprüfung](prerequisite-checker.md).  

##  <a name="BKMK_Security"></a> Voraussetzungsprüfungen für Sicherheitsrechte  
Die folgende Tabelle listet Prüfungen auf, die die Voraussetzungsprüfung für Sicherheitsrechte ausführt.

|Ausgeführte Prüfung|Erklärung|Schweregrad|Standortanwendbarkeit|
|---|---|---|---|
|**Administratorrechte für einen Standort der zentralen Verwaltung**|Damit wird überprüft, ob das Benutzerkonto, unter dem das Configuration Manager- Setup ausgeführt wird, auf dem Standortcomputer der zentralen Verwaltung über **Administrator**rechte verfügt. |Fehler|Primärer Standort|
|**Administratorrechte für den erweiterten primären Standort**|Überprüft, ob das Benutzerkonto, das Setup ausführt, am eigenständigen primären Standort, der erweitert wird, über **Administrator**rechte verfügt.|Fehler|Standort der zentralen Verwaltung|
|**Administratorrechte für das Standortsystem**|Überprüft, ob das Benutzerkonto, unter dem das Configuration Manager- Setup ausgeführt wird, auf dem Standortservercomputer über **Administrator**rechte verfügt. |Fehler|Standort der zentralen Verwaltung, <br>Primärer Standort, <br>Sekundärer Standort|
|**Administratorrechte für CAS-Computer am erweiterten primären Standort**|Überprüft, ob das Computerkonto des Standorts der zentralen Verwaltung, am eigenständigen primären Standort, der erweitert wird, über **Administrator**rechte verfügt.|Fehler|Standort der zentralen Verwaltung|
|**Verbindung zu SQL Server am Standort der zentralen Verwaltung**|Überprüft, ob das Benutzerkonto, das das Configuration Manager-Setup am primären Standort ausführt, um einer bereits vorhandenen Hierarchie beizutreten, die Rolle **sysadmin** auf der Instanz des SQL Servers für den Standort der zentralen Verwaltung innehat.|Fehler|Primärer Standort|
|**Administratorrechte für das Computerkonto des Standortservers**|Überprüft, ob das Computerkonto des Standortservers auf dem Computer von SQL Server und dem Verwaltungspunkt über **Administrator**rechte verfügt.|Fehler|Primärer Standort, <br>SQL Server|
|**Kommunikation zwischen Standortsystem und SQL Server**| Hiermit wird überprüft, ob ein gültiger Dienstprinzipalname (Service Principal Name, SPN) in Active Directory-Domänendiensten für das Konto registriert ist, das für die Ausführung des SQL Server-Diensts der SQL Server-Instanz vorgesehen ist, die als Host für die Configuration Manager-Standortdatenbank verwendet wird. Zur Unterstützung der Kerberos-Authentifizierung muss ein gültiger SPN in den Active Directory-Domänendiensten registriert sein.|Warnung|Sekundärer Standort, <br>Verwaltungspunkt|
|**Sicherheitsmodus des SQL Servers**|Überprüft, ob SQL Server für die Windows-Authentifizierungssicherheit konfiguriert ist.|Warnung|SQL Server|
|**Systemadministratorrechte für SQL Server**|Überprüft, ob das Benutzerkonto, unter dem das Configuration Manager-Setup ausgeführt wird, über die Rolle **sysadmin** für die SQL Server-Instanz verfügt, die für die Datenbankinstallation ausgewählt wurde. Bei dieser Überprüfung tritt auch dann ein Fehler auf, wenn vom Setup nicht auf die SQL Server-Instanz zugegriffen werden kann, um die Berechtigungen zu prüfen.|Fehler|SQL Server|
|**Systemadministratorrechte für Referenzstandort für SQL Server**|Überprüft, ob das Benutzerkonto, unter dem das Configuration Manager-Setup ausgeführt wird, über die Rolle **sysadmin** für die SQL Server-Rolleninstanz verfügt, die als Datenbank des Referenzstandorts ausgewählt wurde. **sysadmin**-Rollenberechtigungen für SQL Server sind erforderlich, um Standortdatenbanken zu ändern.|Fehler|SQL Server|

|**Aktives MP-Replikat**|Hiermit wird auf ein aktives Verwaltungspunktreplikat geprüft.|Fehler|Primärer Standort|
|**Administratorrechte für den Verteilungspunkt**|Überprüft, ob das Benutzerkonto, das das Setup ausführt, über **Administrator**rechte für den Verteilungspunktcomputer verfügt.|Warnung|Verteilungspunkt|
|**Administratorrechte für den Verwaltungspunkt**|Überprüft, ob das Computerkonto des Standortservers über **Administrator**rechte für den Verwaltungspunkt- und Verteilungspunktcomputer verfügt.|Warnung|Verwaltungspunkt|
|**Administrative Freigabe (Standortsystem)**|Überprüft, ob die erforderlichen administrativen Freigaben auf dem Standortsystemcomputer vorhanden sind.|Warnung|Verwaltungspunkt|
|**Anwendungskompatibilität**|Hiermit wird überprüft, ob alle aktuellen Anwendungen mit dem Anwendungsschema kompatibel sind.|Warnung|Standort der zentralen Verwaltung, <br>Primärer Standort|
|**BITS aktiviert**|Überprüft, ob BITS (Background Intelligent Transfer Service) auf dem Standortsystemcomputer des Verwaltungspunkts installiert wurde. Fehler bei dieser Prüfung können folgende Ursachen haben: BITS ist nicht installiert, die IIS 6.0-WMI-Kompatibilitätskomponente für IIS 7.0 ist auf dem Computer oder dem Remote-IIS-Host nicht installiert, oder die IIS-Remoteeinstellungen konnten von Setup nicht überprüft werden, weil gemeinsame IIS-Komponenten nicht auf dem Standortservercomputer installiert wurden.|Fehler|Verwaltungspunkt|
|**BITS installiert**|Überprüft, ob BITS in IIS installiert ist.|Warnung|Verwaltungspunkt|
|**Sortierung ohne Beachtung der Groß-/Kleinschreibung bei SQL Server**|Überprüft, ob von der SQL Server-Installation eine Sortierung ohne Beachtung der Groß-/Kleinschreibung verwendet wird (Beispiel: SQL_Latin1_General_CP1_CI_AS).|Fehler|SQL Server|
|**Eigenständigen primären Standort auf vorhandene Versions- und Standortcodes prüfen**|Überprüft, ob der erweiterte primäre Standort ein eigenständiger primärer Standort ist und die gleiche Version von Configuration Manager, aber einen anderen Standortcode als der zu installierende zentrale Standort aufweist.|Fehler|Standort der zentralen Verwaltung, <br>Primärer Standort|
|**Auf nicht kompatible Sammlungsverweise prüfen**|Bei einem Upgrade überprüft diese Prüfung, ob Sammlungen nur auf Sammlungen desselben Typs verweisen.|Fehler|Standort der zentralen Verwaltung|  
|**Clientversion auf einem Verwaltungspunktcomputer**|Damit wird überprüft, ob Sie den Verwaltungspunkt auf einem Computer installieren, auf dem keine andere Version des Configuration Manager-Clients installiert ist.|Fehler|Verwaltungspunkt|
|**Konfiguration der Speichernutzung durch SQL Server**|Überprüft, ob für SQL Server ein Speicherlimit konfiguriert wurde. Sie sollten einen Maximalwert für den SQL Server-Speicher festlegen.|Warnung|SQL Server|
|**Dedizierte Instanz von SQL Server**|Überprüft, ob eine dedizierte Instanz von SQL Server zum Hosten der Configuration Manager-Standortdatenbank konfiguriert wurde. Wenn die Instanz von einem anderen Standort verwendet wird, müssen Sie eine andere Instanz für den neuen Standort auswählen. Alternativ können Sie den anderen Standort deinstallieren oder dessen Datenbank zu einer anderen SQL Server-Instanz verschieben.|Fehler|Standort der zentralen Verwaltung, <br>Primärer Standort, <br>Sekundärer Standort|
|**Auf dem Server vorhandene Komponenten des Configuration Manager-Servers**|Damit wird überprüft, ob auf dem für die Installation des Standorts ausgewählten Computer bereits ein Standortserver oder eine Standortsystemrolle installiert wurde.|Fehler|Standort der zentralen Verwaltung, <br>Primärer Standort, <br>Sekundärer Standort|
|**Firewallausnahme für SQL Server**|Überprüft, ob die Windows-Firewall deaktiviert ist bzw. ob eine relevante Windows-Firewallausnahme für SQL Server eingerichtet wurde. Erlauben Sie einen Remotezugriff auf sqlservr.exe bzw. die TCP-Ports. Standardmäßig wird TCP-Port 1433 von SQL Server überwacht, und TCP-Port 4022 wird vom SQL-Server-Dienst-Broker (SSB) verwendet.|Fehler|Standort der zentralen Verwaltung, <br>Primärer Standort, <br>Sekundärer Standort, <br>Verwaltungspunkt|
|**Firewallausnahme für SQL Server (eigenständiger primärer Standort)**|Überprüft, ob die Windows-Firewall deaktiviert ist bzw. ob eine relevante Windows-Firewallausnahme für SQL Server eingerichtet wurde. Erlauben Sie einen Remotezugriff auf sqlservr.exe bzw. die TCP-Ports. Standardmäßig wird TCP-Port 1433 von SQL Server überwacht, und TCP-Port 4022 wird vom SQL-SSB verwendet.|Warnung|Primärer Standort (nur eigenständig)|
|**Firewallausnahme bei SQL Server für Verwaltungspunkt**|Überprüft, ob die Windows-Firewall deaktiviert ist bzw. ob eine relevante Windows-Firewallausnahme für SQL Server eingerichtet wurde.|Warnung|Verwaltungspunkt|
|**IIS-HTTPS-Konfiguration**|Überprüft IIS-Websitebindungen für das HTTPS-Kommunikationsprotokoll. Wenn Sie die Installation von Standortrollen, für die HTTPS erforderlich ist, vorgenommen haben, konfigurieren Sie die IIS-Websitebindungen auf dem angegebenen Server mit einem gültigen Public-Key-Infrastructure-Serverzertifikat (PKI).|Warnung|Verwaltungspunkt, <br>Verteilungspunkt|
|**IIS-Dienst wird ausgeführt**|Überprüft, ob IIS installiert wurde und auf dem Computer ausgeführt wird, auf dem der Verwaltungs- oder Verteilungspunkt installiert werden soll.|Fehler|Verwaltungspunkt, <br> Verteilungspunkt|
|**Sortierung des erweiterten primären Standorts vergleichen**|Überprüft, ob die Standortdatenbank für den eigenständigen primären Standort, den Sie erweitern, die gleiche Sortierung aufweist wie die Standortdatenbank am Standort der zentralen Verwaltung.|Fehler|Standort der zentralen Verwaltung|
|**Microsoft Remote Differential Compression-Bibliothek (RDC) registriert**|Überprüft, ob die RDC-Bibliothek auf dem Configuration Manager-Standortserver registriert ist.|Fehler|Standort der zentralen Verwaltung, <br>Primärer Standort, <br>Sekundärer Standort|
|**Microsoft Windows Installer**|Überprüft die Version von Windows Installer. |**Microsoft XML Core Services 6.0 (MSXML60)**|Überprüft, ob auf dem Computer MSXML Version 6.0 oder höher installiert ist.|Warnung|Standort der zentralen Verwaltung, <br>Primärer Standort, <br>Sekundärer Standort, <br>Configuration Manager-Konsole, <br>Verwaltungspunkt, <br>Verteilungspunkt|
|**Mindestversion von .NET Framework für die Configuration Manager-Konsole**|Hiermit wird überprüft, ob Microsoft .NET Framework 4.0 auf dem Computer mit der Configuration Manager-Konsole installiert ist. |**Remoteverbindung mit WMI am sekundären Standort**|Überprüft, ob von Setup eine Remoteverbindung mit WMI auf dem sekundären Standortserver hergestellt werden kann.|Warnung|Sekundärer Standort|
|**Erforderliche SQL Server-Sortierung**|Überprüft, ob die SQL Server-Instanz und die Configuration Manager-Standortdatenbank (falls vorhanden) zur Verwendung der Sortierung „SQL_Latin1_General_CP1_CI_AS“ konfiguriert sind. ##  <a name="BKMK_Requirements"></a> Voraussetzungsprüfungen für Systemanforderungen
